# Полный сценарий работы 2CAN35 Sportage

Цель: держать штатный mode1 от автора, иметь mode2 для обновления и mode3 для
лабораторной работы: CAN read/write + Raise UART rx/tx. Этот документ нужен,
чтобы завтра не повторять хаотичные проверки.

## Что сейчас считаем рабочей базой

- Mode1: прошивка автора под Sportage, сейчас ветка `04.35.00.08`.
- Mode2: штатный update-режим, вход физически кнопкой или программной командой.
- Mode3: наш lab-режим `firmware/canlog/2can35_04350008_mode3_lab_can_uart_usb.bin`
  с CAN RX/TX и Raise UART RX/TX.
- Локальный пульт: `dashboard/server.py`, URL `http://127.0.0.1:8765/`.

```bash
cd /Users/legion/Downloads/2CAN35_CANBOX_WORK_2026-05-08/canbus2can35_repo
```


## Подключение в машине

1. Адаптер подключен к USB MacBook.
2. CAN-M и CAN-C подключены как в штатной схеме.
3. UART магнитолы можно оставить подключенным, если проверяем штатную работу
   mode1. Для чистого CAN/UART lab лучше понимать, что именно подключено:
   магнитола напрямую или через внешний Raise.
4. В TEYES выбран профиль Raise Kia/Hyundai Sportage 17-18, если проверяем
   UART/Raise совместимость.

## Переключение режимов

В dashboard есть блок `Режимы адаптера`:

- `Mode 1 canbox` - возврат в штатную прошивку автора.
- `Mode 2 update` - update-режим для загрузки bin через штатный протокол.
- `Mode 3 lab` - CAN+UART лаборатория.

Если программный переход не отвечает, физический fallback:

1. Полностью обесточить адаптер.
2. Зажать кнопку.
3. Подать питание.
4. Ориентироваться по пищалке: 1 писк mode1, 2 писка update, 3 писка lab.

## Как снимать чистый лог

1. В dashboard нажать `Live CAN+UART`.
2. Слева выбрать одно действие из `Сет на завтра`.
3. Нажать `Старт записи`.
4. Сделать только это действие 3-5 раз.
5. Нажать `Стоп и анализ`.
6. Проверить верхние кандидаты.
7. Нажать `Закрепить`, если кандидат выглядит правильно.

Файл закреплений:

```text
data/learned_assignments.jsonl
```

Raw-лог текущей live-сессии пишется в:

```text
logs/live/
```

## Приоритетные доборы

### 1. RCTA / предупреждение слева-справа

Текущий статус: `0x58B` виден, но в наших попытках оставался
`11 00 04 00 00 00 00 20`.

Нужный тест:

1. Включить зажигание.
2. Включить `R`.
3. Создать реальную помеху слева сзади 10-15 секунд.
4. Создать реальную помеху справа сзади 10-15 секунд.
5. Ничего больше не нажимать.

### 2. Парктроники

Текущий статус: `0x4F4` даёт статус/SPAS, `0x436` был всегда `00 00 00 00`.

Нужный тест:

1. `R` активен.
2. Препятствие заднее левое: далеко, средне, близко.
3. Заднее центр: далеко, средне, близко.
4. Заднее правое: далеко, средне, близко.
5. То же для передних датчиков, если работают без движения.

### 3. Подогрев руля / заднее стекло через StarLine

Нужный тест:

1. Один лог только `подогрев руля on`.
2. Один лог только `подогрев руля off`.
3. Один лог только `обогрев заднего стекла on`.
4. Один лог только `обогрев заднего стекла off`.

Не смешивать с климатическими кнопками на панели.

### 4. UART магнитолы

Нужный тест:

1. В TEYES включить USB music, BT music, FM, AM, CarPlay/Android Auto.
2. На каждом источнике менять трек/станцию.
3. В навигации включить маневр, улицу, ETA, дистанцию.
4. Снимать `Live CAN+UART`, чтобы видеть и UART, и CAN одновременно.

## Быстрая отправка на приборку

Dashboard блок `Приборка` отправляет повтором:

- source/music/track;
- FM;
- nav TBT;
- clear.

Это не доказывает правильность M-CAN кадров напрямую. Это проверка маршрута:

```text
dashboard -> USB CDC / serial -> адаптер / UART path -> TEYES или canbox logic
```

Для прямой проверки M-CAN использовать технический raw CAN TX только с
включенным `Разрешить TX` и только по заранее согласованному ID.

## Что уже подтверждено

- Люк: C-CAN `0x541`, `DATA[7] & 0x02`.
- Задний ход: лучший рабочий набор кандидатов `0x111`, `0x169`, `0x50E`, `0x53E`;
  для подачи +12V в mode1 используется логика прошивки автора.
- Парковка/SPAS: `0x4F4` лучший статусный кандидат.
- Raise UART:
  - `0x05` двери/капот/багажник, bit `0x40` под люк пока эксперимент.
  - `0x03` климат popup.
  - `0x04` радар.
  - `0x09` HU source/media status.
  - `0x7D` reverse local.
  - `0x7F` версия.
