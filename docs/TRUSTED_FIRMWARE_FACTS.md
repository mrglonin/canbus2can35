# Проверенная база по 2CAN35 Sportage

Этот файл - единственный короткий источник истины. Все DBC-кандидаты,
непроверенные профили, старые гипотезы по `0x114/0x197`, `RCTA`, `TPMS`,
`ADAS`, `SVM`, `mode3` и чистой C-прошивке убраны из рабочей документации.

## 1. Штатная модель автора

Штатная прошивка автора работает как обычный canbox и как USB-update loader.

| Режим | Что известно |
|---|---|
| canbox | Основной режим. Адаптер читает CAN автомобиля, физический задний ход и UART/USB входные команды, затем сам отправляет нужные данные в CAN автомобиля и/или магнитолу. |
| update | Штатный режим обновления по USB. Используется APK/утилитой обновления. |

Важно: APK не обязан отправлять навигацию и музыку как raw CAN. Для прошивки
автора подтверждена другая модель: APK/магнитола отправляет команды в USB/API
или Raise/HU формате, а прошивка уже конвертирует их в M-CAN.

## 2. Подтвержденная прошивка-основа

Рабочий файл автора, от которого строился текущий мод:

```text
37FFDA054247303859412243_04350008.bin
```

Подтвержденная строка профиля в прошивке:

```text
HYK-RZ-10-0001-VK
```

UID адаптера:

```text
37 FF DA 05 42 47 30 38 59 41 22 43
```

## 3. Подтвержденный вход HU/Raise

Семья команд источника:

```text
FD .. 09 <source> ...
```

Подтвержденные примеры:

| Событие | Кадр |
|---|---|
| FM | `FD 08 09 02 00 65 46 00 BE` |
| USB music | `FD 0A 09 16 00 00 00 00 02 00 2B` |
| Bluetooth music | `FD 06 09 11 00 00 20` |
| Navigation source | `FD 06 09 06 00 00 15` |
| AUX | `FD 06 09 12 00 00 21` |
| Media off | `FD 06 09 80 00 00 8F` |

Статус: это вход от HU/магнитолы/canbox-протокола в логику прошивки, не
гарантированная команда для отображения на магнитоле при отправке обратно.

## 4. Подтвержденный USB/API -> M-CAN path

В дизассембле v08 подтверждены эти переходы:

| USB/API command | M-CAN ID | Роль |
|---:|---:|---|
| `0x20` | `0x4E8` | FM/station text transport |
| `0x21` | `0x4F6` | media/source text transport в v08 |
| `0x22` | `0x490` | USB/track text transport |

Критичная правка: использовать `0x4E6` как главный generic media ID больше
нельзя. В v08 прошивки автора для команды `0x21` подтвержден `0x4F6`.

Адреса в прошивке:

```text
0x080057EC        media/source parser
0x080057F0        чтение source byte
0x080057F6        сравнение с предыдущим source
0x080057FE..5804  previous source USB 0x16 -> clear/send M-CAN 0x490
0x08005842..584C  previous source radio 0x02 -> clear/send M-CAN 0x4E8

0x08005458        cmd 0x20 -> 0x4E8
0x08005464..546C  cmd 0x21 -> 0x4F6
0x0800546E..5476  cmd 0x22 -> 0x490
```

## 5. Подтвержденные чтения CAN автомобиля

Эти пункты можно использовать как рабочую базу приложения.

| Функция | Шина/ID | Данные | Статус |
|---|---|---|---|
| Передняя левая дверь | C-CAN `0x541` | `DATA[1] bit0` | подтверждено |
| Передняя правая дверь | C-CAN `0x541` | `DATA[4] bit3` | подтверждено |
| Багажник | C-CAN `0x541` | `DATA[1] bit4` | подтверждено |
| Капот | C-CAN `0x541` | `DATA[2] bit1` | подтверждено |
| Задняя левая дверь | C-CAN `0x553` | `DATA[3] bit0` | подтверждено |
| Задняя правая дверь | C-CAN `0x553` | `DATA[2] bit7` | подтверждено |
| Задний ход | C-CAN `0x169` | `DATA[0] & 0x0F == 0x07` | подтверждено |
| Скорость | C-CAN `0x316` | `DATA[6]` | подтверждено рабочей прошивкой |
| Температура двигателя | C-CAN `0x329` | `(DATA[1] - 0x40) * 0.75` | подтверждено |
| Наружная температура | C-CAN `0x044` | `(DATA[3] - 0x52) / 2` | подтверждено |

Не переносить сюда без новой проверки:

- TPMS;
- RCTA/RCCW/BCW exact bits;
- ADAS/SVM exact periods;
- CarPlay/Android Auto raw M-CAN отправку;
- `0x114/0x197` как обязательную модель отправки медиа;
- clean-C замену штатной прошивки.

## 6. Текущий мод v20

`v20` не заменяет модель автора. Он только добавляет sideband к штатной
прошивке v08.

Файл для USB update:

```text
firmware/trusted/v20/20_v08_mode1_v20_USB.bin
```

Команды:

| Команда | Что делает |
|---:|---|
| `0x70` | включить/выключить raw CAN stream |
| `0x76` | прочитать один raw CAN кадр из ring buffer |
| `0x77` | прочитать decoded snapshot |
| `0x78` | одноразово отправить raw CAN кадр |
| `0x79` | V20 health/capabilities без CAN-подключения |

Ограничение: `0x78` не воспроизводит штатную медиа/навигационную логику. Для
медиа и навигации приложение должно говорить с прошивкой в ее родном USB/API
формате.
